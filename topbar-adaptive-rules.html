AI 分析顶部操作栏自适应规则

适用范围：全屏 AI 分析对话顶部栏。目标是文本先合理压缩，按钮保持固定点击区，放不下时从右到左收进“更多”。

最终规则

1. 对话区最小宽内容区最小 380px。小于该值时不再压缩，外层展示横向滚动条。
2. 项目名称最小 42px，最大 320px，按内容宽度自适应，超出省略。
3. 历史会话触发器最小 74px，最大 320px；其中会话名至少显示 3 个中文汉字。
4. 纯 Icon 断点顶部栏宽度 ≤700px 时，除历史会话外，右侧文字按钮统一变纯 Icon。
5. 操作按钮间距Icon 按钮固定 32×32px，按钮间距固定 8px，不随宽度继续压缩。
6. 更多收起仍放不下时，从右到左逐个收进“更多”；收起后一级栏不再展示该按钮。

收起顺序

一级栏原始顺序收起顺序更多面板展示顺序
历史会话 / 新会话 / 分享 / 小窗 / 用量 / 项目记忆 项目记忆 → 用量 → 小窗 → 分享 → 新会话 新会话 → 分享 → 小窗 → 用量 → 项目记忆

历史会话不进入更多；它是当前上下文入口，保留在一级栏。

宽度阶段

条件表现
W > 700px文字按钮正常展示：新会话、分享、小窗。
W ≤ 700px右侧操作统一转纯 Icon；历史会话仍展示文本。
纯 Icon 后仍溢出从右到左收进更多，直到不溢出。
内容区 < 380px停止压缩内容，展示横向滚动条。

二级面板

面板规则
历史会话面板宽度 min 200px / max 320px，在对话区域内定位，不超出内容可视区。
更多面板只展示已收起按钮；图标与名称完整保留；顺序与一级栏原顺序一致。
项目记忆面板在对话区域内打开，避免越出屏幕或遮挡到不可操作区域。

实现参考

const ICON_ONLY_BREAKPOINT = 700;
const MIN_CONTENT_WIDTH = 380;

const textItems = {
  title: { min: 42, max: 320 },
  conversation: { min: 74, max: 320 },
};

const actionOrder = ['new', 'share', 'window', 'usage', 'memory'];
const collapseOrder = [...actionOrder].reverse();

function layoutTopbar() {
  setIconOnly(topbarWidth <= ICON_ONLY_BREAKPOINT);
  resetCollapsedActions();

  for (const action of collapseOrder) {
    if (!isOverflowing()) break;
    moveToMore(action);
  }
}
