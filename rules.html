输入框底部操作区自适应规则

返回交互 Demo

输入框底部操作区用于承载数据源入口、技能入口、计划、模型选择、字数状态和发送等操作。该区域需根据容器宽度、按钮数量和输入状态进行自适应展示，避免操作项互相挤压或影响发送链路。

基础展示

  1. 底部操作区采用单行布局，左右分区展示。
  2. 左侧优先展示数据源入口、技能按钮及其他扩展能力入口。
  3. 右侧固定承载字数状态（按原显示规则展示）、模型选择入口和发送按钮。
  4. 发送按钮始终保持可见，不参与折叠。
  5. 模型选择入口在底部空间足够时展示完整模型名称；空间不足时优先降级为纯图标。

技能按钮规则

  1. 技能按钮可展示为「图标 + 文案」形式。
  2. 技能按钮不设置固定最大宽度；只要底部有可用空间，就完整展示文案。

计划按钮规则

  1. 常规状态下，计划按钮可展示为「图标 + 文案」。
  2. 当字数状态出现或可用空间不足时，计划按钮可降级为纯图标展示。
  3. 计划能力仍需保持可访问，不应因降级而丢失操作入口。

字数状态规则

  1. 当输入内容达到展示条件时，需展示字数状态，例如 4800/5000。
  2. 字数状态优先级高于普通扩展按钮。
  3. 字数状态出现后，操作区需先收起模型名称；仍不足时，再压缩左侧按钮的展示形式。

扩展按钮规则

  1. 新增按钮不默认套用统一展示规则。
  2. 每个新增按钮需单独判断是否需要展示文案、是否仅展示图标、是否支持折叠到更多菜单。
  3. 判断依据包括：按钮含义是否仅靠图标可识别、是否属于高频操作、是否影响核心流程、是否存在下拉配置内容。
  4. 对于智能报告模板这类扩展能力，可根据空间情况独立展示，也可收纳至更多菜单，并通过二级菜单展示具体模板列表。具体规则上线时根据功能形态再决定。

溢出与更多菜单

  1. 当按钮数量增加或字数状态占用空间导致操作区不足时，低优先级扩展能力可收纳至更多菜单。
  2. 更多菜单中的能力需保留原有操作完整性。
  3. 存在二级选项的能力，例如报告模板，应支持在更多菜单内继续展开选择。
  4. 下拉菜单需设置最大宽度，长文本省略展示，避免菜单超出输入框或页面可视区域。

当前线上规则基准

  1. 不展示字数状态时，技能和计划按钮可完整展示。
  2. 展示字数状态时，计划按钮可降级为纯图标。
  3. 模型选择入口优先展示完整模型名称，空间不足时先于左侧按钮降级为纯图标。
  4. 发送按钮始终固定可见。
  5. 后续新增按钮需在此规则基础上补充单独的展示策略。
